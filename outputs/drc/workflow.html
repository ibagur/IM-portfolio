Workflow Guide — Ebola DRC 2026 Data Repository

Data sources for the 2026 Bundibugyo Ebolavirus outbreak, harmonised to 519 DRC MoH health zones

Join key: nom  ·  519 health zones  ·  data/shapefiles/DRC_Health_zones.shp
Contents

For Data Consumers

You do not need to run any scripts. When you pull the repository, all processed outputs and the merged build are already committed and up to date. Scripts are only needed by contributors who update raw source files.

Recommended outputs for analysis

Mapping & GIS
build/drc_health_zones.geojson

All 25 vector layers embedded per zone. Load directly into QGIS, geopandas, R sf, or a web map.

Tabular analysis
build/long/<dataset>__<metric>.csv

Long-format CSV — one row per zone (per date for time-series). Canonical zone names already resolved.

Per-dataset (pre-build)
data/<dataset>/processed/*.csv

Same data as build/long/ but per-dataset. Useful before a build has been run locally.

OD matrices (travel, mobility, displacement)
data/<dataset>/processed/*.matrix.csv

Not in the GeoJSON. Use qa/matrix_log.csv as the catalog, then load the raw CSVs directly.

Never use raw/ for analysis. Raw files are upstream sources in their original format (PDFs, Excel workbooks, NetCDF rasters, shapefiles). The processed/ folder is the clean, standardised, zone-name-resolved version ready for use.

Update frequency guide

🔴 Outbreak-linked (frequent) epi/, insp_sitrep/ — updated frequently during the active outbreak.
🟡 Static socioeconomic (annual) worldpop/, gdp_pc/, ccvi/, fao_lccs/ — only when a new upstream version is released.
🟢 Infrastructure (infrequent) grid3_healthsites/, healthsites_io/, refugee_sites/, osrm/ — on new GRID3 version or significant OSM update in DRC.
🟠 Mobility matrices (ad hoc) IDP/, flowminder/ — per IOM DTM round or Flowminder report; no fixed schedule.

ACLED conflict: placeholder at province grain. Zone-level event data requires a separate event-level pull — tracked in GitHub Issues.

For Contributors — Running Scripts When Raw Data Is Updated

Scripts only need to be re-run when the raw source files in raw/ change. All Python commands use pipenv with Python 3.11. R scripts run with your local R installation.

Standard contributor flow

1
Update files in data/<dataset>/raw/
Binary files are tracked via Git LFS — run git lfs install once if you haven't.
2
Update metadata.yaml — at minimum set retrieved_on to today
3
Run the dataset's process script (see below)
4
pipenv run python -m tools.qa <dataset>
Fix any failures before proceeding. Exit code is non-zero if anything fails.
5
pipenv run python -m tools.build_geojson
Only needed if you changed a vector dataset. Matrix-only changes don't require a rebuild.
6
pipenv run python -m pytest tests/
7
Open a PR — CI runs QA + tests automatically and blocks merge on failure

Python process scripts

py data/cross-border-movements/process.py ▶

Re-run when: new Imperial College report or corrected poe_coordinates.csv.

pipenv run python data/cross-border-movements/process.py

Output: processed/cross_border__poe_passengers__static.csv — columns: nom, n_poes, mean_daily_passengers, mean_weekly_passengers, poe_names

py data/flowminder/process.py ▶

Re-run when: new Flowminder OD matrix files placed in raw/.

pipenv run python data/flowminder/process.py

Outputs: processed/flowminder__{inflow,outflow}__static.matrix.csv (519×519 OD matrices) + zone_resolution_log.csv (diagnostic)

py data/IDP/process.py ▶

Re-run when: new IOM DTM matrix files placed in raw/.

pipenv run python data/IDP/process.py

Outputs: idp__individuals__static.matrix.csv, idp__individuals__monthly.matrix.csv, idp__individuals__weekly.matrix.csv

py data/refugee_sites/process.py ▶

Re-run when: raw OSM GeoJSON refreshed from Overpass/HOT.

pipenv run python data/refugee_sites/process.py

Output: processed/refugee_sites__sites__static.csv — nom, sites

Pipeline tools

# Validate all datasets (required before PR; CI also runs this)
pipenv run python -m tools.qa

# Rebuild the merged GeoJSON (only after vector dataset changes)
pipenv run python -m tools.build_geojson

# Run unit tests
pipenv run python -m pytest tests/

Cutting a release (maintainers only)

Requires gh CLI authenticated and no QA failures.

pipenv run python -m tools.release
git add build/ qa/qa_log.csv qa/matrix_log.csv qa/reports/ README.md
git commit -m "New build YYYY-MM-DD"
git push
